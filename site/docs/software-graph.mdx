---
title: "Software Graph"
description: "The stable semantic contract used by Ontoly frontends, queries, plugins, MCP, and Skills."
---

The Software Graph is Ontoly's canonical output.

It is a versioned JSON document containing repository metadata, nodes, edges,
diagnostics, indexes, statistics, provenance, and extension metadata.

## Node families

The alpha graph models TypeScript and framework concepts such as:

- packages and modules
- functions and methods
- classes and interfaces
- type aliases and enums
- routes and controllers
- services and providers
- configuration and environment variables
- events and resources

## Relationship families

Relationships are directed, typed, deterministic graph edges. Common examples:

- `IMPORTS`
- `EXPORTS`
- `CONTAINS`
- `CALLS`
- `DEPENDS_ON`
- `USES`
- `READS`
- `WRITES`
- `IMPLEMENTS`
- `EXTENDS`
- `HANDLES`
- `MOUNTS`
- `INJECTS`
- `AUTHORIZES`
- `REGISTERED_IN`

## Stable IDs

Stable IDs allow graph output to survive rebuilds, support snapshot tests, and
make graph diffing possible.

```text
module:src/auth/service.ts
fn:src/auth/service.ts:login
class:src/auth/user-service.ts:UserService
route:POST:/login
model:User
```

## Serialization

The alpha serialization format is canonical JSON. Binary formats are deferred
until the graph specification is stable.

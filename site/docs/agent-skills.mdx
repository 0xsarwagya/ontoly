Ontoly exposes software understanding through MCP capabilities and portable
Agent Skills.

The Skill layer teaches workflow. Ontoly provides the knowledge.

## Standard workflow

Every official Skill teaches the same core behavior:

1. Verify that an Ontoly graph exists.
2. Build one with `ontoly build .` if it is missing.
3. Check graph trust and diagnostics.
4. Use Ontoly MCP capabilities.
5. Inspect repository files only when the graph cannot answer.
6. Cite graph evidence and confidence.
7. Explain fallbacks clearly.

## MCP

Start MCP from a source checkout:

```sh
pnpm ontoly mcp --list
pnpm ontoly mcp
```

Representative capabilities include:

- `GraphStatistics`
- `ExplainModule`
- `FindDependencies`
- `ImpactAnalysis`
- `TraceExecution`
- `FindConfigurationUsage`
- `FindAuthenticationFlow`
- `FindAuthorization`
- `FrameworkReport`

## Skills

Official Skills live in the repository under `skills/`:

- architecture review
- impact analysis
- codebase onboarding
- request tracing
- dependency analysis
- security review
- configuration analysis
- framework analysis
- documentation
- refactoring
- performance analysis
- dead-code analysis
- migration analysis
- SDK generation

Validate them:

```sh
pnpm skills:validate
pnpm skills:validate-installed
```

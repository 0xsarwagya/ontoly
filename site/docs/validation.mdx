---

Ontoly is evaluated by evidence, not feature count.

The validation infrastructure measures:

- graph generation
- node and edge coverage
- semantic understanding
- trust
- diagnostics
- determinism
- skill behavior
- package health
- documentation links
- performance
- regressions

## Run validation

```sh
pnpm validate
pnpm evaluate
pnpm benchmark:performance
```

Run release gates:

```sh
pnpm release:gates
```

## Outputs

Validation outputs are committed under `validation/` and `reports/`:

- validation summaries
- semantic leaderboards
- regression baselines
- skill reports
- benchmark reports
- release gate reports
- publish readiness reports

## Release rule

A release should not proceed when semantic coverage regresses, trust decreases,
unexpected diagnostics appear, performance exceeds thresholds, Skill validation
fails, package validation fails, or graph determinism changes unexpectedly.

# Ontoly 0.1.0-alpha.15 Public Preview Release Notes

Prepared: 2026-07-15

## Status

Prepared for GitHub Actions publication. Use the publish workflow result as the
source of truth for npm package publication.

## Highlights

- Stabilizes Ontoly's TypeScript-native Software Graph pipeline for the first
  public preview.
- Freezes the core Software Graph, Query Engine, Semantic Index, Enhancer,
  Capability, MCP, CLI, and Agent Skill surfaces for public-preview testing.
- Ships deterministic validation infrastructure for graph quality, semantic
  understanding, agent workflows, package metadata, skills, installed artifacts,
  and performance.
- Keeps graph generation, semantic retrieval, evidence packs, impact analysis,
  and implementation planning bounded and reproducible.
- Provides human inspection through generated Software Graph HTML and Mermaid
  artifacts without making visualization the product.

## Architecture

Ontoly remains organized around one stable pipeline:

```text
Repository
  -> Compiler
  -> Software Graph
  -> Semantic Index
  -> Enhancers
  -> Capabilities
  -> CLI / MCP / Skills / Plugins
```

The compiler owns graph construction. Enhancers and capabilities consume
immutable artifacts and produce deterministic outputs. Skills remain workflow
guidance only; they do not contain compiler or query logic.

## Semantic Index

- Natural-language intent resolution maps requests to stable graph nodes before
  running impact, evidence, or planner workflows.
- Evidence generation is bounded, ranked, and deduplicated so agent-facing
  workflows do not serialize full graphs.
- Alias and vocabulary generation are deterministic and repository-scoped.

## Enhancers

- The Enhancer API defines deterministic artifact transforms over existing
  Ontoly artifacts.
- Enhancers do not parse repositories, mutate the Software Graph, or call AI.
- Artifact metadata, caching, manifests, validation, and pipeline visualization
  are available for extension authors.

## Capabilities

- Capability execution is graph-first and budgeted.
- Long-running workflows return bounded partial results instead of hanging.
- Confidence is derived from graph evidence and diagnostics, not guessed.
- Evidence Pack is the standard compact context artifact for agent workflows.

## MCP

- MCP capabilities validate inputs before execution.
- Errors are structured and include actionable diagnostics.
- Capability names and response schemas are stable for public-preview
  validation.

## Agent Skills

- Official skills are independently installable and validate both source and
  installed artifacts.
- Skills require Ontoly graph/MCP usage first and repository search only as a
  fallback.
- LLM-facing Ontoly workflows require LLM Enhancement guidance.

## Validation

The public preview is backed by:

- Semantic Evaluation Harness.
- Validation Lab.
- Agent Workflow Validation.
- Skills Validation and Installed Skill Validation.
- Package Metadata Validation.
- Documentation link and Markdown checks.
- Performance and stress profiles.

## Benchmarks

Public Preview benchmark evidence is generated by the validation lab and
semantic evaluation tooling. Final benchmark numbers should be taken from the
GitHub publish workflow and post-publish smoke tests.

Current local Public Preview evidence shows semantic evaluation, stress, skills, package,
docs, typecheck, tests, build, validation lab, release gates, and npm pack
dry-runs passing. The earlier validation-lab performance failure was
investigated without refreshing the baseline; it was not reproducible under a
serial rerun, and a separate real semantic-index artifact write-path slowdown
was fixed.

## Known Limitations

- The current public distribution is still TypeScript-first.
- Framework analyzers are intentionally conservative and prioritize evidence
  quality over broad inference.
- npm package smoke tests should be rerun after publishing the Public Preview artifacts.
- The public preview is an API-freeze checkpoint, not a promise that no
  naming changes will occur before stable v1.

## Migration Notes

From `0.1.0-alpha.14` to `0.1.0-alpha.15`:

- No Software Graph schema migration is expected.
- No compiler behavior migration is expected.
- No new node or edge kinds are introduced by this release-readiness pass.
- Consumers should pin compatible package versions and rerun validation against
  their repositories after upgrading.
- Agent Skill users should reinstall skills from the Public Preview commit or package so
  installed artifacts match source validation.

## Recommended Release Checklist

- Push the public-preview commit to GitHub.
- Run the GitHub npm publish workflow with the intended dist-tag.
- Run npm install smoke tests against the published packages.
- Create the GitHub Release using these notes, updated with final workflow evidence.
